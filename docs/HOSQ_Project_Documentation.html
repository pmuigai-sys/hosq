1. System Overview

The Hospital Queue Management System (HOSQ) is a comprehensive real-time solution designed to streamline the patient experience and hospital staff workflows. Built on a serverless architecture using Supabase and Africa's Talking, it ensures maximum scalability and reliability in the African market.

Key Features:

  • Patient Portal: Self-service registration and live queue tracking.
  • Staff Dashboard: Role-based access for Doctors, Billing, and Pharmacists.
  • Admin Panel: Full control over staff, stages, and emergency priority.
  • Smart SMS: Automated notifications via Africa's Talking.

2. Technical Architecture

HOSQ uses a modern decentralized stack to ensure high performance even on mobile networks.

Component Technology
Frontend React 18 + TypeScript + Vite
Real-time Communication Supabase Realtime (Cdc)
Backend Logic Supabase Edge Functions (Deno)
Database PostgreSQL (hosted on Supabase)
SMS Integration Africa's Talking REST API

3. Setup and Installation

To deploy HOSQ in a new environment, follow these steps:

  1. Environment Configuration: Create a .env.local file with your Supabase URL and Anon Key.
  2. Database Initialization: Run the provided SQL migrations to set up tables and triggers.
  3. SMS Setup: Set your Africa's Talking API Key and Username as Supabase Secrets.
  4. Deployment: Deploy the Edge Functions using supabase functions deploy send-sms.

4. User Guide for Hospital Staff

Staff members can manage the queue flow using the following actions:

  • Call: Notifies the next patient to proceed to the counter.
  • Next: Completes the current stage and moves the patient to the next department.
  • Emergency Flag: Prioritizes a patient, moving them to position #1.
