## Release Process
1. **Develop**: All features developed on feature branches.
2. **Review**: Peer review by project owner Peter Thairu Muigai.
3. **Build**: Run `npm run build` to verify the production bundle.
4. **Tag**: Create a new Git tag using semantic versioning.
5. **Deploy**: Push to the production branch to trigger CI/CD.
